# exercise_3 当前代码说明

这个文件夹目前保存的是第三次积分赛自控小车的“任务层 + 地图规划层”代码。当前重点是先把题目 1、题目 2 的自动导航逻辑搭起来，还没有接真实电机、IMU、编码器和 ToF 驱动。

## 当前已完成

目前已经有一套 4x4 栅格地图和 BFS 路径规划骨架：

- 小车当前位置记录：`x / y / dir`
- 16 个格子的访问记录：`visited`
- 方格边阻塞记录：`blocked`
- 方格边是否已探测记录：`known`
- BFS 找最近未访问格
- BFS 规划到指定格子
- 任务 1：从 A 进入并从 C 口驶出
- 任务 2：遍历 16 个格子后从 C 口驶出

注意：目前任务流程里的运动和传感器函数还是占位版本，代码能表达逻辑，但不能直接驱动真实小车。

## 文件说明

```text
task.h
```

任务层对外接口。其他文件如果想启动比赛任务，只需要包含这个头文件。

主要内容：

- `TaskResult`
- `task_1_go_from_a_to_c()`
- `task_2_traverse_16_cells_then_exit_c()`

```text
taskc.c
```

任务流程实现。

它负责把 BFS 规划出来的方向转换成任务动作：

```text
检测当前格周围边
规划下一步方向
转向
前进一格
更新当前位置
重复直到任务完成
```

当前其中的这些函数还是占位：

- `motion_turn_left_90()`
- `motion_turn_right_90()`
- `motion_turn_back_180()`
- `motion_move_one_cell()`
- `motion_stop()`
- `sensor_front_blocked()`
- `sensor_left_blocked()`
- `sensor_right_blocked()`

后续需要把它们替换成真实底层代码。

```text
nav_map.h
```

地图与 BFS 规划层的对外接口。

主要提供：

- 地图结构体 `RobotNav`
- 方向枚举 `Direction`
- 转向枚举 `TurnAction`
- 地图初始化
- 标记访问
- 设置边阻塞
- 更新坐标
- BFS 规划函数

```text
nav_map.c
```

地图与 BFS 规划层的具体实现。

它只处理 4x4 离散格子，不直接控制电机，也不直接读取传感器。

核心函数：

- `nav_init()`
- `nav_set_edge_blocked()`
- `nav_update_position()`
- `nav_plan_to_nearest_unvisited()`
- `nav_plan_to_cell()`

## 当前坐标假设

当前代码暂定场地坐标为：

```text
x 从左到右：0, 1, 2, 3
y 从上到下：0, 1, 2, 3

(0,0) (1,0) (2,0) (3,0)
(0,1) (1,1) (2,1) (3,1)
(0,2) (1,2) (2,2) (3,2)
(0,3) (1,3) (2,3) (3,3)
```

当前起点和出口假设：

```text
A 口进入后：小车先执行“入图动作”，到达 (0,2)，车头朝东
C 口对应格：小车先到 (3,1)，再朝东驶出
```

如果后续实测发现入口或出口对应格子不同，需要修改：

- `nav_map.c` 中 `nav_init()` 的初始 `x / y / dir`
- `taskc.c` 中 `EXIT_C_X / EXIT_C_Y`

## 当前任务流程

### 任务 1

入口函数：

```c
TaskResult task_1_go_from_a_to_c(void);
```

流程：

```text
执行入图动作
到达起始格 (0,2)
BFS 规划到 C 口对应格子
转向东
前进一格驶出
```

### 任务 2

入口函数：

```c
TaskResult task_2_traverse_16_cells_then_exit_c(void);
```

流程：

```text
执行入图动作
到达起始格 (0,2)
while 16 格没有全部访问：
    读取前/左/右 ToF 结果
    更新当前格的 blocked 边
    BFS 找最近未访问格
    转向 BFS 给出的下一步方向
    前进一格
    更新当前位置
遍历完成后规划到 C 口
驶出 C 口
```
